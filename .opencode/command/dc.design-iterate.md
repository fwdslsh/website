# Visual Web Design & Iteration Workflow

## Roles & Responsibilities

- **@parallel-work-orchestrator**: Plans and tracks design tasks
- **@web-design-expert**: Reviews design, UX, and provides design feedback
- **Chrome MCP Tools**: Visual verification and interaction testing

## Workflow Steps

### 1. Initial Design Implementation
- @parallel-work-orchestrator creates design tasks
- primary agent assigns @svelte5-expert-dev UI/styling tasks
- @svelte5-expert-dev implements components with styling
- Capture initial screenshots using Chrome MCP tools

### 2. Visual Design Review
- @web-design-expert reviews screenshots and code
- Evaluates: visual hierarchy, color theory, typography, spacing, layout
- Provides specific design feedback and recommendations
- @parallel-work-orchestrator updates tasks based on feedback

### 3. UX & Interaction Review
- Capture screenshots of different states (hover, focus, active, error)
- @web-design-expert evaluates user flows and interactions
- Reviews accessibility, responsive behavior, and touch targets
- Checks brand consistency and design system adherence
- @parallel-work-orchestrator tracks UX improvements

### 4. Implementation of Feedback
- @svelte5-expert-dev implements design improvements
- Updates CSS, animations, transitions, and interactive states
- Captures new screenshots showing changes

### 5. Visual Verification
- Use Chrome MCP tools to:
  - Capture full-page and component screenshots
  - Test responsive breakpoints
  - Verify interactive states
  - Check accessibility contrast ratios
  - Verify buttons and other elements are not clipped or hidden by the smaller viewports
- Compare before/after screenshots
- Validate design consistency across pages/components

### 6. Iteration Loop
Repeat steps 2-5 until:
- ✓ Visual design meets quality standards
- ✓ UX is engaging and intuitive
- ✓ Accessibility requirements met
- ✓ Responsive design works across breakpoints
- ✓ Brand consistency maintained
- ✓ All interactive states polished
