Getting Started with unify

unify is a modern static site generator that uses familiar Apache Server-Side Includes (SSI) syntax with modern tooling. Build maintainable sites without learning new templating languages or managing complex build pipelines.

⚡ Quick Start

Get a site running in under 2 minutes with zero configuration.

Installation

unify requires Bun for optimal performance. If you don't have Bun installed:

bash
# Install Bun
curl -fsSL https://bun.sh/install | bash

# Install unify globally
bun add -g @fwdslsh/unify

# Verify installation
unify --version
⚠️ Alternative Runtimes

While unify works with Node.js and Deno, Bun provides the best performance with native APIs.

Create Your First Site

Start with the official starter template:

bash
# Clone the starter template
git clone https://github.com/fwdslsh/unify-starter my-site
cd my-site

# Install dependencies
bun install

# Start development server
bun run dev

Your site will be available at http://localhost:3000 with live reload enabled.

Manual Setup

Prefer to start from scratch? Create a minimal project structure:

bash
# Create project directory
mkdir my-unify-site && cd my-unify-site

# Create basic structure
mkdir src src/.components src/assets
touch src/index.html src/.components/header.html src/.components/footer.html

Basic Project Structure

A typical unify project follows this structure:

text
my-site/
├── src/                    # Source files
│   ├── .components/        # Reusable components
│   │   ├── header.html
│   │   ├── footer.html
│   │   └── nav.html
│   ├── assets/            # Static assets
│   │   ├── css/
│   │   ├── js/
│   │   └── images/
│   ├── index.html         # Homepage
│   ├── about.html         # Additional pages
│   └── blog/              # Nested directories
│       └── index.md       # Markdown content
├── dist/                  # Generated output
└── package.json           # Dependencies (optional)

Your First Component

Create a reusable header component in src/.components/header.html:

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Site</title>
    <link rel="stylesheet" href="/assets/css/style.css">
</head>
<body>
    <header>
        <nav>
            <a href="/">Home</a>
            <a href="/about">About</a>
            <a href="/blog">Blog</a>
        </nav>
    </header>

Create a footer component in src/.components/footer.html:

html
    <footer>
        <p>&copy; 2024 My Site. Built with unify.</p>
    </footer>
</body>
</html>

Using Server-Side Includes

Now create your homepage in src/index.html using Apache SSI syntax:

html
<!--#include virtual="/.components/header.html" -->

<main>
    <h1>Welcome to My Site</h1>
    <p>Built with unify - static sites for people who remember HTML.</p>
    
    <section>
        <h2>Features</h2>
        <ul>
            <li>No learning curve - uses Apache SSI syntax</li>
            <li>Live reload development server</li>
            <li>Markdown support with frontmatter</li>
            <li>Component-based architecture</li>
        </ul>
    </section>
</main>

<!--#include virtual="/.components/footer.html" -->

CLI Commands

unify provides a simple command-line interface:

Command Description Options
unify build Build your site to dist/ --pretty-urls, --base-url
unify serve Start development server --port, --host
unify watch Watch files and rebuild --verbose

Build for Production

bash
# Basic build
unify build

# Build with pretty URLs (removes .html extensions)
unify build --pretty-urls

# Build with custom base URL
unify build --base-url https://mysite.com

# Build and serve locally to test
unify build && unify serve --port 8080

Development Workflow

The typical development workflow with unify:

  1. Start development server: unify serve
  2. Edit files: Changes auto-reload in browser
  3. Add components: Create reusable parts in .components/
  4. Build for production: unify build
  5. Deploy: Upload dist/ to any static host

Key Benefits

  • Zero learning curve: If you know HTML and Apache SSI, you know unify
  • Fast builds: Powered by Bun's native APIs for maximum performance
  • Live development: Server-Sent Events for instant browser updates
  • Portable output: Pure HTML/CSS/JS that works anywhere
  • Secure by default: Path traversal protection and input validation
  • Framework-free: No client-side JavaScript frameworks required
