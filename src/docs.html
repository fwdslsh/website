Documentation

Get started with Unify in just a few steps.

Installation

You can install Unify via npm. For a quick try, you don't even need to install globally — just use npx as shown below.

$ npm install -g @fwdslsh/unify
$ unify build
$ unify serve

The commands above will install Unify globally, build the site from the default src/ directory into dist/, and start a local dev server. Alternatively, run npx @fwdslsh/unify serve in any project to serve it without a global install.

Project Structure

By default, Unify looks for a src/ folder with your source files and outputs to dist/. Within src, you can organize reusable components and layouts in special directories:

my-site/
├── src/
│    ├── .components/ (partials like header, footer, etc.)
│    ├── .layouts/ (HTML layouts for pages)
│    ├── css/ (your CSS files)
│    ├── index.html (example page)
│    └── about.md (example Markdown page)
└── dist/ (output directory)

Using Includes & Layouts

Create reusable pieces of HTML like headers, footers, or sidebars in .components/, and reference them in your pages or layouts using an include comment. For example:

<!--#include virtual="/.components/header.html" -->
<main>
  <h1>Welcome!</h1>
</main>
<!--#include virtual="/.components/footer.html" -->

You can also use a special <slot> element in layouts to define where page content goes, and template variables (like {{ title }}) to inject frontmatter data into layouts.

Commands Overview

  • unify build [options] – Compile the site (with options for source directory, output directory, pretty URLs, etc.).
  • unify serve [options] – Start a dev server on localhost (use -p to specify port) with live reload.

Run unify --help to see all available commands and flags.
