Getting Started with unify

unify is a modern static site generator that uses familiar Apache Server-Side Includes (SSI) syntax with modern tooling. Build maintainable sites without learning new templating languages or managing complex build pipelines.

‚ö° Quick Start

Get a site running in under 2 minutes with zero configuration.

Installation

unify requires Bun for optimal performance. If you don't have Bun installed:

bash
# Install Bun
curl -fsSL https://bun.sh/install | bash

# Install unify globally
bun add -g @fwdslsh/unify

# Verify installation
unify --version
‚ö†Ô∏è Alternative Runtimes

While unify works with Node.js and Deno, Bun provides the best performance with native APIs.

Create Your First Site

Start with the official starter template:

bash
# Clone the starter template
git clone https://github.com/fwdslsh/unify-starter my-site
cd my-site

# Install dependencies
bun install

# Start development server
bun run dev

Your site will be available at http://localhost:3000 with live reload enabled.

Manual Setup

Prefer to start from scratch? Create a minimal project structure:

bash
# Create project directory
mkdir my-unify-site && cd my-unify-site

# Create basic structure
mkdir src src/_includes src/assets
touch src/index.html src/_includes/header.html src/_includes/footer.html

Your First Component

Create a reusable header component in src/_includes/header.html:

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Site</title>
    <link rel="stylesheet" href="/assets/css/style.css">
</head>
<body>
    <header>
        <nav>
            <a href="/">Home</a>
            <a href="/about">About</a>
            <a href="/blog">Blog</a>
        </nav>
    </header>

Create a footer component in src/_includes/footer.html:

html
    <footer>
        <p>&copy; 2024 My Site. Built with unify.</p>
    </footer>
</body>
</html>

Using Server-Side Includes

Now create your homepage in src/index.html using Apache SSI syntax:

html
<!--#include virtual="/_includes/header.html" -->

<main>
    <h1>Welcome to My Site</h1>
    <p>Built with unify - static sites for people who remember HTML.</p>
    
    <section>
        <h2>Features</h2>
        <ul>
            <li>No learning curve - uses Apache SSI syntax</li>
            <li>Live reload development server</li>
            <li>Markdown support with frontmatter</li>
            <li>Component-based architecture</li>
        </ul>
    </section>
</main>

<!--#include virtual="/_includes/footer.html" -->

Slot System with Fallbacks

Unify's slot system supports fallback content when no content is provided:

html
<!-- Layout with fallback content -->
<header>
  <div data-slot="header">
    <h1>Default Site Title</h1>
  </div>
</header>

<main>
  <div data-slot="content"></div>
</main>

Pages can override the fallback content by providing slot content:

html
<!-- Page that overrides header slot -->
<template slot="header">
  <h1>Custom Page Title</h1>
  <nav>...</nav>
</template>

<p>This content goes into the content slot</p>

Slot Injection in Includes

Include files can also use slots for dynamic content:

html
<!-- _includes/card.html -->
<div class="card">
  <h3 data-slot="title">Default Title</h3>
  <div data-slot="content">Default content</div>
</div>

Use the include with custom data attributes:

html
<include src="/_includes/card.html" 
         data-title="My Card Title"
         data-content="Custom card content" />

CLI Commands

unify provides a simple command-line interface:

Command Description Options
unify build Build your site to dist/ --pretty-urls, --base-url
unify serve Start development server --port, --host
unify watch Watch files and rebuild --verbose

Build for Production

bash
# Basic build
unify build

# Build with pretty URLs (removes .html extensions)
unify build --pretty-urls

# Build with custom base URL
unify build --base-url https://mysite.com

# Build and serve locally to test
unify build && unify serve --port 8080

Development Workflow

The typical development workflow with unify:

  1. Start development server: unify serve
  2. Edit files: Changes auto-reload in browser
  3. Add components: Create reusable parts in _includes/
  4. Build for production: unify build
  5. Deploy: Upload dist/ to any static host

Layout Discovery

Unify provides flexible layout discovery options to structure your pages:

Automatic Layout Discovery

Place a _layout.html file in any directory to automatically apply it to all pages in that directory and subdirectories:

html
<!-- src/_layout.html -->
<!DOCTYPE html>
<html>
<head>
  <div data-slot="head"></div>
</head>
<body>
  <div data-slot="content"></div>
</body>
</html>

Explicit Layout Reference

Use the <link rel="layout"> element to specify a custom layout:

html
<!-- In your HTML page -->
<html>
<head>
  <link rel="layout" href="/custom-layout.html">
  <template slot="head">
    <title>My Custom Page</title>
  </template>
</head>
<body>
  <h1>Page content goes here</h1>
</body>
</html>

Short Name Layout Discovery

Create named layouts in _includes/ with the pattern _[name].layout.html:

html
<!-- src/_includes/_blog.layout.html -->
<!DOCTYPE html>
<html>
<head>
  <div data-slot="head"></div>
</head>
<body>
  <header>Blog Layout</header>
  <div data-slot="content"></div>
</body>
</html>

Reference it using the short name:

html
<link rel="layout" href="blog">

Key Benefits

  • Zero learning curve: If you know HTML and Apache SSI, you know unify
  • Fast builds: Powered by Bun's native APIs for maximum performance
  • Live development: Server-Sent Events for instant browser updates
  • Portable output: Pure HTML/CSS/JS that works anywhere
  • Secure by default: Path traversal protection and input validation
  • Framework-free: No client-side JavaScript frameworks required
